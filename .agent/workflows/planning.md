# 기획 에이전트 (Planning Agent)

사용자가 만들고 싶은 기능을 간단하게 입력하면, 안티그래비티(AI)가 최적의 실행을 할 수 있도록 **완벽한 구현 프롬프트**를 작성합니다.

## 실행 방법

사용자가 `/planning [아이디어]` 형태로 입력합니다.
예시: `/planning 퀴즈에 타이머 기능 추가하고 싶어`

## 워크플로우 단계

### 1단계: 프로젝트 컨텍스트 수집
- `progress.md`를 읽어 프로젝트 구조, 기술 스택, 현재 상태를 파악합니다.
- 관련된 기존 코드 파일들을 빠르게 스캔합니다.

### 2단계: 아이디어 분석 및 확장
사용자의 간단한 입력을 분석하여 다음을 추론합니다:
- 이 기능이 어떤 문제를 해결하는지
- 기존 코드의 어떤 부분과 연관되는지
- 필요한 기술적 구성요소는 무엇인지

### 3단계: 최적 프롬프트 생성
아래 형식으로 **완벽한 구현 프롬프트**를 작성하여 사용자에게 제시합니다:

```markdown
## 🎯 기능 정의
[기능의 명확한 정의와 목적]

## 📁 수정/생성할 파일
| 파일 | 작업 | 역할 |
|------|------|------|
| 파일명 | 수정/신규 | 이 파일에서 할 일 |

## 🏗️ 기술 요구사항
- 사용할 기술 스택 (Supabase, Cloudflare Worker, YouTube API 등)
- DB 스키마 변경 사항 (필요시)
- API 엔드포인트 추가/수정 (필요시)

## 🎨 UI/UX 요구사항
- 화면 레이아웃 및 디자인 가이드
- 반응형 대응
- 다크모드/라이트모드 고려

## ⚠️ 예외 처리 & 주의사항
- 에지 케이스와 에러 핸들링
- 기존 기능과의 충돌 여부
- 보안 고려사항

## ✅ 검증 계획
- 테스트 방법 및 확인 사항
```

### 4단계: 사용자 확인
- 생성된 프롬프트를 사용자에게 보여주고 수정 요청을 받습니다.
- 사용자가 "좋아", "진행해" 등으로 승인하면 해당 프롬프트를 바로 실행합니다.

## 주의사항
- 사용자는 비전공자이므로 기술 용어는 쉽게 풀어서 설명합니다.
- 프로젝트의 기존 패턴(Supabase DB → Cloudflare Worker API → 프론트엔드)을 따릅니다.
- 한국어로 작성합니다.
- 단계별 분할 (Task Splitting): 한 번에 모든 코드를 짜라고 하지 마라. 대신 1단계(데이터베이스), 2단계(서버), 3단계(화면)로 나누어 순차적으로 실행할 수 있게 가이드를 줘라.
- 의존성 체크: "이 기능을 구현하기 위해 먼저 수정되어야 할 파일이 있는지 우선순위를 명시하라."
