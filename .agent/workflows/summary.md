# Summary 에이전트

프로젝트의 **'단기 기억력'** 역할을 합니다. 작업이 끝날 때마다 `progress.md` 파일에 현재 상태를 압축적으로 업데이트하여, 다음 세션에서 AI가 바로 작업을 이어받을 수 있게 합니다.

## 실행 방법

사용자가 `/summary` 라고 입력합니다.

## 워크플로우 단계

### 1단계: 현재 대화 분석
- 이번 대화에서 진행된 코드 수정, 파일 생성/삭제, 이슈 해결 등을 파악합니다.
- 변경된 파일들의 diff를 확인합니다.

### 2단계: 기존 progress.md 확인
- `progress.md` 파일이 프로젝트 루트(`./progress.md`)에 있는지 확인합니다.
- 있다면 기존 내용을 읽어 업데이트할 준비를 합니다.
- 없다면 새로 생성합니다.

### 3단계: progress.md 업데이트
아래 형식으로 `progress.md`를 **전체 덮어쓰기** 합니다. 항상 최신 상태만 유지합니다:

```markdown
# 📋 프로젝트 진행 상황
> 마지막 업데이트: [날짜 시간]

## 🔑 현재 맥락 (AI가 이어받기 위한 핵심 정보)
- [현재 프로젝트가 어떤 상태인지 2-3줄로 압축]
- [진행 중이던 작업이 있다면 어디까지 했는지]
- [반드시 알아야 할 설정이나 환경 정보]

## ✅ 최근 완료된 작업
1. [가장 최근 작업 - 쉬운 말로]
2. [그 전 작업]
3. [그 전 작업]
(최대 5건, 오래된 것은 삭제)

## ⚠️ 현재 이슈 / 알려진 버그
- [해결되지 않은 문제가 있다면 기록]
- [없으면 "현재 알려진 이슈 없음"]

## 📋 다음에 할 일
1. [우선순위 높은 것부터]
2. [그 다음]
3. [그 다음]

## 📁 최근 변경된 파일
- `파일명` — 무엇을 바꿨는지 한 줄 설명
```

### 4단계: 사용자에게 요약 보고
- 업데이트된 내용을 사용자에게 **비전공자가 읽기 쉬운 형태로** 간단히 보여줍니다.
- 예: "오늘 유튜브 재생 버그를 고쳤고, 다음에는 타이머 기능을 추가하면 됩니다!"

## 핵심 규칙
1. **압축적으로**: 전체 분량이 길어지지 않게 핵심만 기록합니다.
2. **쉬운 말로**: 코드 용어 대신 "~하는 기능", "~화면" 등 일상어를 사용합니다.
3. **항상 최신만**: 오래된 완료 작업은 5건을 초과하면 삭제합니다.
4. **맥락 중심**: "현재 맥락" 섹션이 가장 중요합니다. 다음 세션에서 이것만 읽어도 바로 작업을 이어받을 수 있어야 합니다.
5. **한국어로**: 모든 내용을 한국어로 작성합니다.
6. **토큰절약모드**: 소스 코드를 직접 progress.md에 복사해 넣지 마라. 대신 로직의 흐름만 설명해라.
